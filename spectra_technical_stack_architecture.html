# Spectra AI Architect - Technical Stack & Architecture ## 1. Frontend: React (SPA) * **Framework:** React 18+ with TypeScript. * **Styling:** Tailwind CSS for the "Kinetic Precision" design system. * **State Management:** TanStack Query (React Query) for API synchronization and model monitoring. * **UI Components:** Headless UI / Radix UI for accessible, high-performance interactive elements. * **Visualizations:** Recharts or D3.js for GPU metrics and throughput velocity graphs. ## 2. Backend: Java Spring Boot * **Framework:** Spring Boot 3.x. * **Security:** Spring Security with JWT for system operator authentication. * **Processing Pipeline:** * **Extraction:** Integration with Apache POI (DOCX), Jackson (JSON/XML), and PDFBox/Tika (PDF). * **Services:** `TextCleanerService` for OCR and Unicode normalization. * **Vector Database:** Integration with ChromaDB (API v2) via a dedicated Java client or REST templates. * **Orchestration:** Spring Batch or Quartz for managing long-running fine-tuning jobs. ## 3. Machine Learning Interface * **Local Inference:** Ollama API for embeddings (`nomic-embed-text`) and PHI-3 for Q&A generation. * **Training Interface:** JNI or ProcessBuilder to interface with HuggingFace PEFT (LoRA) or Unsloth GPU scripts. * **Model Export:** Automated triggers for `convert_hf_to_gguf.py` upon training completion. ## 4. API Endpoints (Core) * `POST /api/v1/datasets/upload`: Document ingestion. * `GET /api/v1/monitor/system`: Live GPU/CPU utilization metrics (SSE or WebSocket). * `POST /api/v1/training/start`: Initiate LoRA fine-tuning. * `POST /api/v1/playground/query`: RAG-enabled chat interface. * `GET /api/v1/evaluation/compare`: Side-by-side model metrics.
